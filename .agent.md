# Project Agent Rules: expo-android-project-3

## 📋 Project Summary
- **Name**: Traffic Webcams (webcams-trafico)
- **Framework**: Expo (SDK 54) / React Native
- **Router**: Expo Router (file-based)
- **Styling**: NativeWind (Tailwind CSS)

## 🛠️ Technology Stack
- **React Native**: 0.81.5
- **React**: 19.1.0
- **NativeWind**: 4.2.1
- **TypeScript**: 5.9.2
- **OpenSpec**: Specification-driven development framework.

## 📂 Project Structure
- `/app`: Application routes and navigation.
- `/components`: Reusable UI components.
- `/openspec`: Specifications and change management (OPSX).
- `.agent/skills`: Agent-specific knowledge and rules.
- `/scripts`: Utility scripts (e.g., `fetch-webcams.ts`).

## ✍️ Coding and Style Rules
1.  **Styling**: Use **ONLY** Tailwind classes via the `className` prop (NativeWind). The usage of the `style` prop for CSS application is **STRICTLY FORBIDDEN** across the entire project. Avoid `StyleSheet.create` and inline styles.
2.  **Images**: Use `expo-image` for optimized loading.
3.  **List Performance**: Follow rules defined in `.agent/skills/vercel-react-native-skills` (avoid anonymous functions in `renderItem`, use memoization).
4.  **Workflow**: Use `OpenSpec` commands (`/opsx-new`, `/opsx-apply`, etc.) to implement new features in a structured way.
5.  **File Naming**: React component files must use kebab-case naming. For example, if a component is named `SubmitButton`, the filename must be `submit-button.tsx`.
6.  **Components**: Use `Pressable` for all interactive elements instead of `TouchableOpacity` or `TouchableHighlight`. **`TouchableOpacity` is STRICTLY FORBIDDEN** to ensure consistent performance and modern standards. Use NativeWind `active:opacity-*` classes for press feedback. Avoid using the `style` prop with `pressed` state; stick to Tailwind classes.
7.  **Package Management**: ALWAYS use `npx expo install <package-name>` instead of `npm install` for library installation. This ensures that the installed packages are compatible with the current Expo SDK version (SDK 54) and prevents native version mismatches.

## 🚀 Useful Commands
- `npx expo install <package-name>`: Use this to install any package to ensure version compatibility.
- `npx expo install --check`: Validates package alignment with the SDK version and helps fix warnings.
- `npm run fetch-webcams`: Updates camera data from external source.
- `npm run android`: Automatically launches the emulator and the app.
